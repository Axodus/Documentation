## Purpose

ACS can support Marketplace by reviewing listings, summarizing seller quality, checking claims, drafting listing copy, flagging sensitive categories, summarizing disputes, and preparing reports.

## Agent Roles

- Morpheus: ecosystem positioning, listing narrative clarity, governance alignment, user education language, and documentation coherence.
- Agent Smith: adversarial listing review, scam or overclaim detection, seller abuse detection, locked reward policy risk, and dispute or fraud risk.
- Trinity: treasury, fee, settlement, Trading, DeFi, revenue, and liability reporting context.
- Coder: documentation pages, navigation, Markdown structure, and docs scaffolding.

## Workflow

1. Listing or marketplace request is received.
2. Context is loaded from Marketplace, Business, Academy, Tokenomics, Treasury, Governance, and Risk documentation.
3. Morpheus reviews positioning and ecosystem fit.
4. Agent Smith reviews claims, abuse risk, and policy conflicts.
5. Trinity reviews financial, token, Trading, or DeFi claims where relevant.
6. Human or domain reviewer validates output.
7. Governance, treasury, security, or BBA review occurs if required.
8. Output is used for listing, report, or handoff.

## Boundaries

ACS cannot approve sensitive listings as final authority, settle disputes, authorize refunds, move funds, validate legal compliance, guarantee seller results, or override governance and treasury policy.
