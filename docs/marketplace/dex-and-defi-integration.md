## Purpose

Marketplace may integrate with DEX and DeFi for payments, swaps, accepted assets, DaaS services, liquidity, subscriptions, and productized DeFi offerings.

## DEX Integration Models

Possible models include swap before purchase, embedded swap checkout if implemented, seller settlement conversion if supported, treasury conversion if policy allows, and accepted asset expansion with or without DEX conversion.

## DeFi Listing Categories

Categories may include DaaS service, vault or strategy access if approved, liquidity service if policy allows, educational DeFi product, and protocol integration service.

## Required Disclosures

DEX checkout requires slippage, price impact, failed transaction, fee, and refund complexity disclosures. DeFi listings require no-yield-guarantee language, smart contract risk, protocol risk, liquidity risk, and user or treasury exposure context where applicable.

## Design Questions

Questions include accepted assets, swap requirements, slippage responsibility, refund handling after swaps, seller payout asset, treasury exposure, allowed audience, governance review, and security review.
