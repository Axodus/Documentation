## Purpose

ACS supports DEX through token listing review, route analysis, liquidity monitoring, risk summaries, claim review, reporting, and governance brief preparation.

## Agent Roles

Trinity supports liquidity analysis, routing context, Trading integration review, fee and volume reporting, and treasury LP context.

Agent Smith supports adversarial review, scam token risk detection, MEV and execution risk review, unsafe claim detection, and governance bypass detection.

Morpheus supports ecosystem positioning, governance alignment, documentation consistency, and user narrative clarity.

## Workflow

1. DEX request or issue is received.
2. Context is loaded from DEX, DeFi, Tokenomics, Treasury, and Governance docs.
3. Trinity reviews liquidity, route, or trading context.
4. Agent Smith reviews token security, claims, or attack surface.
5. Morpheus reviews positioning and governance alignment.
6. Human or domain reviewer validates.
7. Governance, security, or treasury review is routed if required.
8. Output is used for docs, reports, or handoff.
9. Accountability record is created.

## Boundaries

ACS can analyze, summarize, flag risks, draft reports, prepare handoffs, and suggest safer language. ACS cannot approve token listings, approve treasury liquidity, deploy contracts, guarantee execution or liquidity, override governance, or publish sensitive claims without review.
