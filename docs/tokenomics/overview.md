## Purpose

Axodus Tokenomics defines the economic design, utility, rewards, access, governance participation, and sustainability model for `$Neurons`.

Tokenomics is utility-focused, not investment-focused. It must remain transparent, governance-aware, contract-validated, risk-managed, and accountable.

## Scope

Tokenomics connects Academy, Marketplace, Governance, Business, Treasury, Trading, ACS, and Accountability. It covers token utility, access rules, reward policy, Learn-to-Win rewards, Marketplace utility, governance participation, supply and emission policy, fees if implemented, treasury alignment, contract dependencies, risk language, and reporting.

## Core Principles

- Utility first: token usage should be tied to real ecosystem functions.
- No financial guarantees: documentation must not promise profit, APY, yield, price appreciation, income, or liquidity.
- Sustainability: rewards must be budgeted, reviewable, and reportable.
- Contract truth: public claims must match contract and platform behavior.
- Governance alignment: material tokenomics changes require governance review.
- Anti-abuse: reward farming, sybil behavior, and self-dealing need controls.
- Accountability: reward, fee, treasury, and governance effects should be reportable.

## Canonical Token Notation

Use `$Neurons` as the canonical token notation in new documentation. Historical references to `NRS` should be treated as deprecated or historical unless governance revalidates them.

## Runtime Summary

Tokenomics work should follow a controlled lifecycle: classify the request, run ACS risk and language review when needed, identify affected nuclei, check contract dependencies, assess treasury impact, determine governance requirements, draft reward or utility policy, obtain governance review when material, hand off implementation if approved, create reporting records, and update public documentation with status and caveats.

## Status Terms

Tokenomics mechanisms may be marked as `Draft`, `Planned`, `Experimental`, `Active`, `Needs Contract Validation`, `Governance Required`, `Deprecated`, or `Historical`.
