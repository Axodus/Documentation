## Purpose

This page defines the documentation framing for `$Neurons`, the canonical Axodus ecosystem token notation.

## Definition

`$Neurons` is the Axodus ecosystem token designed to support utility, rewards, access, governance participation, and ecosystem coordination.

It may connect Academy reward flows, Marketplace utility, governance participation, product access, Business services, Trading access rules, and ecosystem incentive models if implementation and governance policy support those uses.

## Canonical Notation

Use `$Neurons` in current documentation. Treat `NRS` as deprecated or historical unless governance revalidates it.

## Token Roles

- Utility: access to products, services, courses, or Marketplace features if supported.
- Rewards: learning or participation rewards under policy.
- Governance participation: possible governance participation asset if defined by Governance and contracts.
- Coordination: shared economic unit connecting Axodus nuclei.
- Marketplace usage: possible payment, discount, credit, or access mechanism if implemented.
- Progression context: token activity may support progression only when policy defines it.

## What `$Neurons` Is Not

`$Neurons` must not be framed as an investment product, guaranteed return product, fixed income instrument, guaranteed yield token, passive income mechanism, risk-free reward, guaranteed liquidity source, or legal/regulatory status claim.

## Historical or Planned References

Historical or planned references mention a 10,000,000 supply, 0.35% burn fee, 0.25% liquidity fee, and 0.5% treasury fee. These values are `Needs Contract Validation` and must not be presented as active unless the production contract and governance records confirm them.

## Lifecycle Concepts

`$Neurons` flows may use lifecycle states such as minted, allocated, earned, issued, locked, unlocked, claimed, spent, burned, and reported. Each state must map to policy and implementation before being treated as active.

## Documentation Requirements

Public documentation should define `$Neurons` before using it, explain utility without investment language, mark contract-dependent features, distinguish locked and unlocked states, include risk language, and verify supply, fees, and contract behavior before making final claims.
