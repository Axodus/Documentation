## Purpose

Lottery transparency depends on reporting that lets users, governance, and reviewers understand tickets, randomness, results, prizes, claims, fees, treasury routing, incidents, and limitations.

## Report Types

Report types may include draw report, ticket report, randomness report, prize report, treasury report, claim report, incident report, and governance record.

## Draw Report Fields

Fields may include draw ID, game model, game model version, status, ticket open and close times, ticket count, ticket root if any, randomness source, request and fulfillment references, draw result, winner count by tier, prize pool, fees, claim window, incidents, governance reference, and limitations.

## Accountability Records

Records should preserve game model approvals, draw configuration, ticket root if used, randomness receipt, prize records, treasury records, and incident records.

## Quality Rules

Reports must use the game model version, match ticket count to finalized tickets, identify randomness source, reconcile prizes and fees, report unclaimed prizes, disclose incidents, state limitations, and avoid exposing private user data.
