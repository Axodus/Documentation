## Purpose

This document standardizes core Axodus terminology for public documentation.

## Approved Terms

- Governance
- Constitutional governance
- DAO federation
- Treasury supervision
- Accountability layer
- Operational runtime
- AI-assisted coordination
- Ecosystem nucleus
- Risk-managed execution

## Discouraged or Prohibited Terms

- Guaranteed profit
- Passive income guarantee
- Risk-free yield
- Infinite APY
- Autonomous unrestricted AI
- Guaranteed token appreciation

These terms may appear only when documentation explicitly says they are prohibited.

## Communication Style

Axodus documentation should be institutional, technical, transparent, educational, and risk-aware.

## Semantic Priorities

- Sustainability
- Transparency
- Governance
- Accountability
- Operational clarity
- Modularity

## Status Language

Use `Active` only for implemented or operationally accepted systems. Use `Draft` for designed but not final systems, `Planned` for intended but not implemented systems, `Experimental` for systems being tested, `Deprecated` for no longer canonical systems, and `Historical Reference` for preserved context.
