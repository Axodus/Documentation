# Scalper Futures

Version: 0.1.0  
Last Updated: 2026-05-16

## Purpose

Scalper Futures is an example concept for a user-facing trading strategy that may be used as an entry product in Axodus Trading.

It is a concept, not an active promise of availability or performance.

## Scope

Scalper Futures is described as a possible short-term futures strategy intended to operate through a user's own exchange account using API keys if implemented.

## Product Activation Purpose

The concept may support user onboarding into Axodus Trading, encourage Academy risk education, connect `$Neurons` utility to product access, and create a path from basic access to advanced services.

## Execution Model

The user creates or uses their own CEX account, creates an API key, disables withdrawals, limits permissions to required trading actions, configures risk parameters if supported, and can pause or revoke the key.

Do not claim support for any exchange until integration is verified.

## Possible Access Rules

Access may depend on `$Neurons` balance, Academy futures risk course completion, Proof of Knowledge, promotional campaign, limited beta, subscription for advanced settings, milestone unlock, or governance-approved policy.

Do not state exact token thresholds or permanent free access unless policy confirms them.

## Futures Risk Disclosure

Futures and leverage can amplify losses and cause liquidation. Funding rates, volatility, exchange outages, API errors, slippage, fees, and unfavorable market regimes can materially affect results.

Scalper Futures does not guarantee profit. Users should only allocate capital they can afford to risk. Axodus does not custody user funds by default. Users remain responsible for exchange accounts, API key permissions, risk settings, and leverage.

## Risk Controls

Possible controls include leverage caps, position size limits, daily loss limits, max drawdown limits, stop loss or exit rules if supported, kill switch, user pause, API error handling, liquidation warnings, funding awareness, and exchange health monitoring.

## Onboarding Requirements

Onboarding should include strategy description, futures risk disclosure, no-profit-guarantee confirmation, Academy risk module if required, API permissions check, withdrawal-disabled confirmation, risk parameter confirmation, and paper or test mode if available.

## Performance Reporting

Reports should include period, realized and unrealized PnL where available, fees, funding, drawdown, trade count, win rate if relevant, liquidation events if any, exchange errors, pauses, limitations, and risk notes. Performance must not be presented as a future guarantee.

## Launch Readiness Gates

Launch requires research, backtest report with limitations, paper trade or limited pilot, API security onboarding, risk disclosure, access policy, monitoring, pause or kill switch, support or incident process, and governance or product approval if required.
