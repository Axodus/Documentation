# AXODUS USER JOURNEY

Version: 0.1 Draft  
Status: Product Journey Model

---

## 1. Overview

The Axodus user journey is designed as a progressive ecosystem exposure model.

The user does not enter through speculation. The user enters through education, utility, Proof of Knowledge, marketplace interaction, NFT-based access and practical product usage.

---

## 2. Core Journey

```txt
Academy
    ↓
Learning to Win
    ↓
Proof of Knowledge
    ↓
Earn $Neurons
    ↓
Marketplace
    ↓
Acquire NFT License
    ↓
Connect NFT to Trading Dashboard
    ↓
Configure Exchange API
    ↓
Use Trading Bot + ACS Agents
    ↓
Generate Initial Revenue
    ↓
Understand Ecosystem Value
    ↓
Become Long-Term Participant / Investor / DAO Member
```

---

## 3. Entry Point

The first user entry point is Academy.

The user must learn before accessing trading automation.

---

## 4. Academy Role

Academy provides:

- free required courses;
- risk education;
- trading basics;
- exchange/API guidance;
- bot operation principles;
- governance context;
- ecosystem orientation;
- quizzes and assessments.

---

## 5. Learning to Win

Learning to Win is the gamified education system where users:

- complete lessons;
- complete quizzes;
- prove knowledge;
- earn $Neurons;
- progress toward certification.

---

## 6. Proof of Knowledge

Proof of Knowledge validates that the user has completed the required learning path before receiving access to advanced products.

PoK should reduce irresponsible access, improve user quality and strengthen ecosystem culture.

---

## 7. Marketplace Role

Marketplace distributes access products such as:

- courses;
- certifications;
- NFT licenses;
- agents;
- MCP subscriptions;
- services;
- compute products.

In this journey, the user uses earned $Neurons to acquire an NFT license that unlocks the Trading Bot.

---

## 8. NFT License

The NFT license is an operational access credential.

It is not a speculative asset by default.

It may define:

- bot access level;
- duration;
- strategy tier;
- maximum capital limit;
- user eligibility;
- renewal rules;
- governance-defined restrictions.

---

## 9. Trading Bot Access

After acquiring the NFT license, the user connects it to the Trading Dashboard.

The dashboard validates:

- wallet;
- NFT ownership;
- certification status;
- eligibility rules;
- product limits.

---

## 10. API-Based Trading

The user connects an exchange API key from their own account.

Principles:

- user capital remains in the user's exchange account;
- API permissions should be limited;
- withdrawals must not be required;
- risk limits must be enforced;
- the free tier must remain highly limited.

---

## 11. ACS Agents

ACS agents assist with:

- strategy configuration;
- risk guidance;
- market condition monitoring;
- parameter recommendations;
- operational warnings;
- user education.

---

## 12. User Maturation

The goal is not immediate investment.

The goal is maturity:

- learn;
- prove;
- access;
- use;
- understand;
- reinvest voluntarily;
- join or create a DAO;
- participate in governance.

---

## 13. Governance Exposure

The user should understand that:

- access rules exist because governance defined them;
- product limits exist because governance protects the ecosystem;
- future product changes may be decided by DAO governance.

---

## 14. Final Principle

Axodus onboarding should create informed participants, not speculative users.
