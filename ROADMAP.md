# Axodus Roadmap

Status: Draft
Version: 0.1.0
Last Updated: 2026-05-16
Owner: Axodus Core

---

## Purpose

This roadmap describes Axodus direction without committing to fixed dates, unverified deployments, or guaranteed outcomes.

## Scope

The roadmap covers documentation, governance, Business, ACS, Academy, Trading, Treasury, product nuclei, security, and accountability. It uses phases instead of deadlines unless dates are confirmed.

## Phase 0 - Documentation Rebuild

- Rebuild repository documentation around VitePress.
- Create `.instructions` for future agents and contributors.
- Separate canonical documentation from historical GitBook material.
- Establish content guardrails and review checklists.

## Phase 1 - Governance and Architecture Stabilization

- Refine constitutional governance.
- Clarify Executive DAO, Boardroom Council, and Community DAO roles.
- Define DAO federation concepts and local autonomy constraints.
- Document proposal and execution workflows.

## Phase 2 - Business Runtime and ACS Alignment

- Define request intake, classification, scope control, and delivery workflow.
- Specify how ACS supports Business and Governance.
- Establish change request and reporting processes.

## Phase 3 - Academy and Token Utility Specification

- Document Learn-to-Win, Proof of Knowledge, tutor validation, and course publishing.
- Clarify `$Neurons` utility without investment claims.
- Define locked and unlocked reward concepts only where contract rules are known.

## Phase 4 - Trading and Treasury Risk Documentation

- Document internal trading infrastructure.
- Document user-facing strategy access and API key security.
- Define treasury policy, risk limits, and reporting expectations.

## Phase 5 - Product Nuclei Expansion

- Expand Mining, Marketplace, DeFi, Lottery, MCP, Compute, and BBA documentation.
- Mark active, planned, experimental, and conceptual systems clearly.

## Phase 6 - Accountability and Public Reporting

- Publish financial report templates.
- Publish governance records.
- Publish release notes and roadmap updates.
- Improve traceability between decisions, execution, and reported outcomes.

## Roadmap Guardrails

This roadmap is directional. It does not promise financial performance, APY, profit, legal status, audit completion, exchange integrations, or final token behavior.
