# DOC-ADR-006 — Documentation Conflict Resolution

## Title

Documentation Conflict Resolution

## Status

PROPOSED

## Context

Portfolio, nucleus, governance, architecture, and public documentation may describe overlapping subjects.

## Problem

No single ordering rule safely resolves every conflict, especially when scope, evidence, approval, and safety boundaries differ.

## Alternatives

1. Prefer the newest document.
2. Prefer the broadest authority level.
3. Prefer a fixed location hierarchy.
4. Resolve by applicable scope, approval, evidence, and the strictest safety boundary.

## Proposed Decision

Use scope-aware resolution with explicit ownership and escalation, always preserving the stricter safety boundary.

## Consequences

Some conflicts remain blocked pending owner decisions rather than being automatically reconciled.

## Open Questions

- Which conflicts require portfolio approval?
- What evidence closes a conflict?
- How will unresolved conflicts affect publication?
