# AXODUS ECOSYSTEM ONBOARDING

Version: 0.1 Draft  
Status: Product Strategy

---

## 1. Objective

Define the onboarding model that introduces users to Axodus through education, utility, marketplace interaction and governed access.

---

## 2. Onboarding Philosophy

Axodus should not onboard users through hype or speculative promises.

Axodus should onboard users through:

- education;
- practical utility;
- visible product function;
- progressive exposure;
- governance awareness;
- real ecosystem interaction.

---

## 3. First Experience

The user begins with a free Academy path.

This path explains:

- market basics;
- trading risk;
- API usage;
- bot limitations;
- governance rules;
- $Neurons utility;
- NFT license mechanics;
- Axodus ecosystem overview.

---

## 4. Required Steps

1. Connect wallet
2. Acquire free Academy course
3. Complete course modules
4. Complete quizzes
5. Earn $Neurons through Proof of Knowledge
6. Use $Neurons in Marketplace
7. Acquire NFT license
8. Unlock Trading Bot Dashboard
9. Connect exchange API
10. Configure risk-limited bot
11. Use ACS guidance
12. Decide whether to continue deeper into the ecosystem

---

## 5. Product Exposure Map

During onboarding, the user interacts with:

- Academy
- Marketplace
- $Neurons
- NFT license
- Trading Dashboard
- ACS Agents
- Governance context

---

## 6. Conversion Philosophy

The user should not be forced to invest.

The user should be invited to continue because the ecosystem demonstrated value.

---

## 7. Free Tier Limits

The initial Trading Bot access should be limited by:

- capital limit;
- duration;
- strategy type;
- leverage limits;
- exchange permissions;
- certification requirements;
- governance-defined controls.

---

## 8. Risk Controls

Onboarding must include:

- risk education;
- clear disclaimers;
- quiz validation;
- API safety instructions;
- no withdrawal permissions;
- capital limits;
- stop conditions;
- responsible-use guidelines.

---

## 9. Success Metrics

Track:

- course completion rate;
- quiz pass rate;
- wallet connection rate;
- NFT license acquisition rate;
- bot activation rate;
- retention;
- user satisfaction;
- support incidents;
- strategy performance;
- reinvestment conversion.

---

## 10. Final Principle

The onboarding process should make the user progressively understand Axodus before asking for deeper commitment.
