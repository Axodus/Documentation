## 1. Responsible Participation Thesis

Lottery participation can create financial and behavioral risk.

CryptoDraw documentation must encourage responsible participation, avoid manipulative language, and make restrictions clear.

## 2. Required Participation Themes

themes:
  no_guaranteed_winnings:
    message:
      - Participation_does_not_guarantee_any_prize_or_profit.

  entertainment_and_probability:
    message:
      - Lottery_outcomes_are_random_and_odds_should_be_understood_before participation.

  jurisdiction_restrictions:
    message:
      - Availability_may_depend_on_user_location_and_applicable_rules.

  age_restrictions:
    message:
      - Participation_may_be_restricted_by_age_or_local_requirements.

  spending_limits:
    message:
      - Users_should_only_participate_with_amounts_they_are_prepared_to lose.

  claim_window:
    message:
      - Winners_must_follow_claim_rules_and_deadlines.

  token_risk:
    message:
      - If_crypto_assets_are_used_their_value_may_change.

  wallet_and_transaction_risk:
    message:
      - Users_are_responsible_for_wallet_transactions_network_fees_and correct addresses.

## 3. User Safety Controls

possible_controls:
  - spending_limits
  - cool_down_periods
  - max_tickets_per_draw
  - self_exclusion_if implemented
  - region_restrictions_if required
  - age_confirmation_if required
  - risk_acknowledgement
  - educational_prompt
  - probability_explainer
  - support_resources_if applicable

## 4. Prohibited User Messaging

prohibited_messaging:
  - guaranteed_winning
  - easy_money
  - passive_income
  - risk_free_lottery
  - investment_lottery
  - play_to_get_rich
  - recover_losses
  - urgency_pressure
  - misleading_odds
  - hiding_costs_or_claim_deadlines

## 5. Public Documentation Guidance

The public Responsible Participation page should:

- state no guaranteed winnings;
- explain jurisdiction and age sensitivity;
- list user safety controls;
- prohibit manipulative messaging;
- link to Risk and Compliance.
