# AXODUS GLOSSARY KNOWLEDGE PACK

Domain: Glossary / Terminology  
Layer: Semantic Memory  
Audience: Agent Coder, ACS agents, documentation agents, governance reviewers, product planners, community writers  
Status: Draft  
Version: 0.1.0  

## Purpose

This folder contains the deep semantic terminology base for the Axodus ecosystem.

These files are not public-facing documentation pages. They are agent-oriented knowledge packs designed to help documentation agents, coding agents, ACS agents, governance reviewers, and future MCP systems use consistent terminology across all Axodus documentation.

The Glossary nucleus is transversal. It does not belong to only one product. It defines the shared vocabulary that connects Governance, Business, Academy, ACS, Trading, Treasury, Tokenomics, Marketplace, DaaS, Security, Accountability, and future products.

## Glossary Thesis

Axodus documentation must use terms consistently.

Inconsistent terms create confusion, governance ambiguity, legal and financial risk, implementation mismatch, and poor user understanding.

The Glossary must help agents and humans understand:

- what a term means;
- where it belongs in the ecosystem;
- what terms should be avoided;
- what terms are still draft or planned;
- what terms require careful financial, legal, technical, or governance language;
- how concepts relate to each other.

The public glossary should be concise and readable.

This knowledge pack can be denser, more explicit, and more operational than the public glossary.

## How Agents Should Use This Pack

Agents must use this pack when writing or reviewing public documentation.

When generating documentation, agents should:

- use the official term when available;
- avoid deprecated aliases;
- preserve capitalization for canonical terms;
- avoid financial/investment framing for `$Neurons`;
- distinguish live, planned, conceptual, and draft terms;
- use governance terms consistently;
- avoid vague or hype-driven DeFi language;
- include cross-links from public glossary entries to related docs;
- mark uncertain implementation details as Draft, Planned, or Needs Review.

## Non-Negotiable Terminology Invariants

- All public-facing glossary content must be written in English.
- `$Neurons` is the canonical token notation.
- Do not use NRS as the primary symbol unless explicitly documenting historical or deprecated references.
- Governance is the coordination heart of Axodus.
- Business is the structured intake and delivery runtime.
- Academy is the education, onboarding, Proof of Knowledge, Learn-to-Win, and token utility layer.
- ACS means AI Coordination System.
- BBA means Blockchain Business & Advertising.
- Locked Rewards must not be described as freely transferable unless implementation confirms it.
- Trading tools must never be described as guaranteed profit systems.
- Treasury must be documented with risk and accountability.
- Agents support decisions but do not replace Governance or human accountability.
- Planned systems must not be described as live systems.
