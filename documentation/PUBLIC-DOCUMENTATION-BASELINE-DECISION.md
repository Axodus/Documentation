# PUBLIC-DOCUMENTATION-BASELINE-DECISION

## Decision

`PASS_WITH_PUBLIC_DOCUMENTATION_STATUS_BASELINE_FROZEN`

## Decision Statement

The public Axodus documentation corpus is frozen for EPIC-05 REQ-01 as an
existing canonical surface that requires coherence remediation before any
expansion workstream.

## Baseline Position

- `Documentation` retains canonical authority for its own public corpus.
- `Institutional` is not promoted to canonical authority transfer.
- `requests/` remains a private execution-governance surface, not a public
  authority surface.
- Physical reorganization is not authorized in this request.
- New product-page expansion is not authorized by default in this request.

## Frozen Public Baseline by Class

| Class | Meaning in this freeze |
| --- | --- |
| `CANONICAL_CURRENT` | Public page can remain authoritative without immediate status repair. |
| `CANONICAL_WITH_QUALIFICATION` | Public page is canonical but still needs tighter qualifiers. |
| `BASELINE_REFERENCE` | Useful reference or directory support, not a final canonical authority target. |
| `ACTIVE_IF_VERIFIED` | Topic may be described publicly, but only evidence can justify active interpretation. |
| `REVIEW_REQUIRED` | Page needs baseline clarification before stronger reliance. |
| `BOUNDARY_REVIEW_REQUIRED` | Page or domain requires disclosure-boundary review before any broadening. |
| `CLAIM_REVIEW_REQUIRED` | Page or domain requires claim-strength review before stronger reliance. |

## Immediate Execution Consequence

REQ-02 must focus on terminology, claims, maturity language, and disclosure
coherence review. It must not begin with bulk rewriting or public-surface
expansion.

## Non-Authorization Clause

This freeze does not authorize:

- new tokenomics claims
- new financial mechanics
- production-readiness claims
- new public operational architecture claims
- migration of Institutional paper content
- public expansion of private request outputs
