# Documentation Ownership

Status: Active Recovery
Version: 0.1.0
Last Updated: 2026-06-04
Owner: Documentation Coordinator

---

## Owner Role

The Documentation Coordinator owns the Documentation nucleus operational process:

- documentation workflow;
- content quality standards;
- validation checklist;
- publication boundary tracking;
- asset inventory;
- drift register;
- recovery reports.

## Reviewers

Reviewers may include:

- affected Nucleus Owner;
- Portfolio Coordinator;
- Safety Reviewer;
- Governance reviewer;
- Documentation QA reviewer;
- technical owner for the documented subsystem.

## Approvers

Approver depends on scope:

| Scope | Required Approval |
|---|---|
| Documentation wording or structure | Documentation Coordinator |
| Nucleus-specific status or behavior | Nucleus Owner |
| Portfolio maturity or roadmap | Portfolio Coordinator |
| Financial/token/treasury/compliance claims | Safety Reviewer and Portfolio Coordinator |
| Governance authority or execution claims | Governance owner/coordinator |
| Publication to official public docs | Documentation Coordinator plus required scope approvers |

## Escalation Path

1. Documentation Coordinator.
2. Affected Nucleus Owner.
3. Safety Reviewer when risk-sensitive.
4. Portfolio Coordinator.
5. Governance/DAO approval when governance policy, treasury, or execution authority is implicated.

## DAO/Governance Relationship

Documentation records governance decisions; it does not create them. Governance or DAO approval must be evidenced before documentation can claim an approved governance outcome, treasury action, protocol execution, or public commitment.

## Drafting vs Publication Authority

Drafting authority allows contributors to prepare candidate material.

Publication authority requires review, approval, validation, and correct publication channel. Drafting does not imply public approval.

## Safety Rule

Documentation cannot infer or grant production execution, wallet signing, treasury movement, trading, swaps, payouts, settlement, credential use, or on-chain writes.
